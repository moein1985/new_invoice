# نقشه راه ویژگی SIP/CTI — شناسایی مشتری با تماس ورودی

## هدف
زمانی که مشتری تماس می‌گیرد، شماره تماس‌گیرنده شناسایی شده و اطلاعات مشتری به‌صورت پاپ‌آپ به کاربر نمایش داده شود.

---

## معماری کلی

```
مشتری ──(تماس)──► IP PBX (Issabel/Asterisk)
                         │
                    داخلی 101 (تلفن فیزیکی) ──► زنگ
                    داخلی 201 (WebSocket) ──► مرورگر کاربر
                         │
                    SIP INVITE + Caller ID
                         │
                    مرورگر (sip.js) ──► شناسایی شماره
                         │
                    tRPC API ──► جستجو در دیتابیس مشتریان
                         │
                    پاپ‌آپ با اطلاعات مشتری
```

### تنظیمات IP PBX (Issabel/Asterisk)
برای اینکه WebSocket کار کند، تنظیمات زیر روی PBX لازم است:
```ini
; /etc/asterisk/http.conf
[general]
enabled=yes
bindaddr=0.0.0.0
bindport=8088
tlsenable=no          ; برای wss: tlsenable=yes
tlsbindaddr=0.0.0.0:8089

; /etc/asterisk/pjsip.conf (یا sip.conf)
; اضافه کردن transport برای WebSocket:
[transport-ws]
type=transport
protocol=ws
bind=0.0.0.0

; ساخت endpoint برای هر داخلی وب:
[201]
type=endpoint
transport=transport-ws
context=from-internal
disallow=all
allow=ulaw,alaw,opus
auth=201
aors=201
```

### امنیت
- رمز SIP به‌صورت رمزنگاری‌شده در دیتابیس ذخیره شود
- ترجیحاً از WSS (WebSocket Secure) استفاده شود
- اتصال SIP فقط از طریق شبکه داخلی (LAN)

### سازگاری با مرورگرها
- Chrome, Edge, Firefox: کامل ✅
- Safari: محدود (WebRTC issues)

## اولویت‌بندی یادداشت‌ها در پاپ‌آپ
یادداشت‌های اسناد (`notes`) به ترتیب زیر نمایش داده می‌شوند:
1. اسناد با وضعیت **در انتظار تایید** (PENDING) — بالاترین اولویت
2. اسناد با وضعیت **رد شده** (REJECTED)
3. اسناد با وضعیت **تایید شده** (APPROVED)
4. مرتب‌سازی ثانویه: جدیدترین اول
