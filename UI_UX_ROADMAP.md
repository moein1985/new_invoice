# نقشه راه بهبود UI/UX

## 🔴 اولویت بالا

### 1. Sidebar Navigation (منوی کناری ثابت)
- **وضعیت فعلی:** کاربر برای رفتن به هر بخش باید به داشبورد برگردد
- **هدف:** منوی کناری ثابت با لینک به: داشبورد، اسناد، مشتری‌ها، تاییدیه‌ها، کاربران، بکاپ
- **جزئیات:**
  - دسکتاپ: sidebar ثابت در سمت راست (RTL) با آیکون و متن
  - تبلت: sidebar جمع‌شونده (فقط آیکون)
  - موبایل: hamburger menu با overlay
  - نمایش آیتم فعال با رنگ متفاوت
  - badge تعداد اعلان‌ها روی آیتم تاییدیه‌ها
- **فایل‌های درگیر:** `app/layout.tsx`، کامپوننت جدید `components/ui/sidebar.tsx`

---

### 2. جایگزینی `confirm()` مرورگر با Modal سفارشی
- **وضعیت فعلی:** حذف مشتری/سند با `confirm()` پیش‌فرض مرورگر انجام می‌شود
- **هدف:** ConfirmDialog component با طراحی هماهنگ با اپلیکیشن
- **جزئیات:**
  - عنوان، پیام، آیکون هشدار
  - دکمه تایید (قرمز برای حذف) و انصراف
  - انیمیشن ورود/خروج
  - بستن با Escape و کلیک بیرون
  - پشتیبانی از انواع: danger (حذف)، warning (هشدار)، info (اطلاع‌رسانی)
- **فایل‌های درگیر:** کامپوننت جدید `components/ui/confirm-dialog.tsx`، هر جایی که `confirm()` استفاده شده

---

### 3. Skeleton Loading به جای متن ساده
- **وضعیت فعلی:** صفحات در حال بارگذاری فقط متن "در حال بارگذاری..." نشان می‌دهند
- **هدف:** skeleton loader (مستطیل‌های خاکستری متحرک) برای هر صفحه
- **جزئیات:**
  - Skeleton برای جدول (ردیف‌های خالی با انیمیشن pulse)
  - Skeleton برای کارت‌های داشبورد
  - Skeleton برای فرم‌ها
  - Skeleton برای صفحه جزئیات سند
  - انیمیشن: `animate-pulse` با gradient
- **فایل‌های درگیر:** کامپوننت جدید `components/ui/skeleton.tsx`، تمام صفحاتی که loading state دارند

---

### 4. Breadcrumb Navigation
- **وضعیت فعلی:** در صفحات عمیق مسیر کاربر مشخص نیست
- **هدف:** نمایش مسیر hierarchical در بالای هر صفحه
- **جزئیات:**
  - مثال: داشبورد > اسناد > سند #۱۲۳ > ویرایش
  - آیتم آخر غیرقابل کلیک (صفحه فعلی)
  - آیتم‌های قبلی لینک‌دار
  - جداکننده: `/` یا `>`
  - پشتیبانی RTL
- **فایل‌های درگیر:** کامپوننت جدید `components/ui/breadcrumb.tsx`، تمام صفحات

## 🟡 اولویت متوسط

### 5. جایگزینی Emoji با آیکون‌های SVG (lucide-react)
- **وضعیت فعلی:** دکمه‌ها از emoji استفاده می‌کنند (✏️، 📄، 📊، 🖨️، ⚙️) که در سیستم‌های مختلف متفاوت رندر می‌شوند
- **هدف:** استفاده از آیکون‌های lucide-react که uniform هستند
- **جزئیات:**
  - ✏️ → `<Pencil />` (ویرایش)
  - 📄 → `<FileText />` (PDF)
  - 📊 → `<Sheet />` (Excel)
  - 🖨️ → `<Printer />` (چاپ)
  - ⚙️ → `<Settings />` (تنظیمات)
  - 👥 → `<Users />` (مشتری‌ها)
  - ⏳ → `<Clock />` (در انتظار)
  - ✅ → `<CheckCircle />` (تایید شده)
  - ❌ → `<XCircle />` (رد شده)
  - ➕ → `<Plus />` (افزودن)
  - 🗑️ → `<Trash2 />` (حذف)
- **فایل‌های درگیر:** تمام صفحات و کامپوننت‌هایی که از emoji استفاده می‌کنند

### 6. نشانگر فیلدهای اجباری
- **وضعیت فعلی:** فرم‌ها هیچ علامتی برای فیلدهای اجباری ندارند، کاربر فقط بعد از submit متوجه می‌شود
- **هدف:** نمایش `*` قرمز کنار label فیلدهای اجباری
- **جزئیات:**
  - ستاره قرمز `*` کنار label
  - متن کمکی زیر فیلد در صورت نیاز
  - رنگ border قرمز برای فیلد خالی بعد از اولین submit
  - پیام خطا زیر فیلد (نه فقط toast)
- **فایل‌های درگیر:** `app/documents/new/page.tsx`، `app/documents/edit/[id]/page.tsx`، `app/customers/page.tsx`

### 7. Real-time Validation (اعتبارسنجی آنی)
- **وضعیت فعلی:** validation فقط موقع submit اجرا می‌شود
- **هدف:** نمایش خطا بلافاصله بعد از blur (خروج از فیلد)
- **جزئیات:**
  - onBlur validation برای هر فیلد
  - نمایش پیام خطا زیر فیلد با انیمیشن fade-in
  - تغییر رنگ border به قرمز
  - آیکون خطا/تایید کنار فیلد
  - فیلدهای عددی: بررسی مقدار min/max
  - فیلد نام مشتری: بررسی خالی نبودن
  - آیتم‌های سند: بررسی نام محصول، تعداد، قیمت
- **فایل‌های درگیر:** `app/documents/new/page.tsx`، `app/documents/edit/[id]/page.tsx`

### 8. فیلترهای فعال قابل مشاهده
- **وضعیت فعلی:** وقتی فیلتر اعمال شده مشخص نیست چه فیلترهایی فعال هستند
- **هدف:** نمایش واضح فیلترهای فعال
- **جزئیات:**
  - chip/badge برای هر فیلتر فعال (مثلاً: "نوع: پیش‌فاکتور ×")
  - دکمه × برای حذف هر فیلتر
  - badge روی دکمه فیلتر با تعداد فیلترهای فعال
  - دکمه "پاک کردن همه" واضح‌تر
  - ذخیره فیلترها در URL (query params) برای اشتراک‌گذاری
- **فایل‌های درگیر:** `app/documents/page.tsx`، `app/customers/page.tsx`

## 🟢 اولویت پایین

### 9. اصلاح Dark Mode
- **وضعیت فعلی:** بعضی کامپوننت‌ها از `bg-white` هاردکد استفاده می‌کنند و در dark mode شکسته به نظر می‌رسند
- **هدف:** پشتیبانی کامل از dark mode در تمام کامپوننت‌ها
- **جزئیات:**
  - جایگزینی `bg-white` با `bg-white dark:bg-gray-800`
  - جایگزینی `text-gray-800` با `text-gray-800 dark:text-gray-200`
  - بررسی تمام border colors
  - بررسی shadow ها در dark mode
  - تست display-settings، فرم‌ها، جداول، کارت‌ها
- **فایل‌های درگیر:** `components/ui/display-settings.tsx`، تمام صفحات

---

### 10. Tooltip برای دکمه‌های آیکونی
- **وضعیت فعلی:** دکمه‌های PDF، Excel، چاپ بدون توضیح هستند
- **هدف:** hover tooltip برای توضیح عملکرد دکمه‌ها
- **جزئیات:**
  - Tooltip component با انیمیشن fade-in
  - موقعیت: بالا/پایین/چپ/راست (auto positioning)
  - تاخیر نمایش: ۳۰۰ms
  - پشتیبانی RTL
  - ARIA: `aria-label` + `role="tooltip"`
- **فایل‌های درگیر:** کامپوننت جدید `components/ui/tooltip.tsx`، دکمه‌های آیکونی

---

### 11. عملیات دسته‌جمعی (Bulk Actions)
- **وضعیت فعلی:** عملیات فقط روی یک آیتم در هر بار امکان‌پذیر است
- **هدف:** انتخاب چندین سند/مشتری و اجرای عملیات همزمان
- **جزئیات:**
  - checkbox در هر ردیف جدول
  - "انتخاب همه" در header
  - نوار عملیات دسته‌جمعی (حذف، اکسپورت، تغییر وضعیت)
  - شمارنده آیتم‌های انتخاب‌شده
  - تایید قبل از عملیات حذف دسته‌جمعی
- **فایل‌های درگیر:** `app/documents/page.tsx`، `app/customers/page.tsx`، router های مربوطه
